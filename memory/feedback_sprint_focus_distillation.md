# Sprint Focus Distillation

When a private project memory defines an active sprint, epic, or operating focus, convert it into routing context rather than copying the private roadmap into public memory.

Private Lacrimosa deployments can keep specific project names, issue chains, provider choices, cost estimates, and dependency ordering in private runtime memory. Public Lacrimosa memory should keep only the reusable behavior:

- Treat active sprint focus as routing input for discovery, triage, content, and implementation specialists.
- Preserve dependency order: spikes and research tasks unblock implementation tasks incrementally.
- Re-check current provider/library status before implementation because automation stacks, pricing, gating, and deprecations change quickly.
- Keep credentials and user verification data in secret storage, never in model context or public reports.
- Distill public notes to decision principles and gotchas, not private roadmaps or issue identifiers.
